# Quick Start Guide - One Thing That Matters Backend

The fastest way to get your Supabase backend up and running.

---

## 5-Minute Setup

### 1. Create Supabase Project (2 minutes)

1. Go to https://app.supabase.com
2. Click "New Project"
3. Name it `one-thing-that-matters`
4. Choose a password and region
5. Wait for provisioning

### 2. Deploy Database Schema (1 minute)

1. In Supabase, open **SQL Editor**
2. Copy entire contents of `supabase_schema.sql`
3. Paste and click **Run**
4. Verify success (green checkmark)

### 3. Configure n8n (2 minutes)

1. Get your keys from Supabase: **Settings** → **API**
2. Add to n8n environment variables:
   - `SUPABASE_URL` = your project URL
   - `SUPABASE_SERVICE_KEY` = your service role key

3. In your n8n workflow, add **HTTP Request** node after LLM:
   - Method: POST
   - URL: `{{$env.SUPABASE_URL}}/rest/v1/watch_candidates?on_conflict=video_id`
   - Headers:
     ```json
     {
       "apikey": "{{$env.SUPABASE_SERVICE_KEY}}",
       "Authorization": "Bearer {{$env.SUPABASE_SERVICE_KEY}}",
       "Content-Type": "application/json",
       "Prefer": "resolution=merge-duplicates"
     }
     ```
   - Body: See `n8n_supabase_config.md` for field mapping

### 4. Test It

Run your n8n workflow, then check Supabase:

```sql
SELECT * FROM watch_candidates ORDER BY created_at DESC LIMIT 5;
```

Done! 🎉

## Most Common Operations

### Get Today's Shortlist

```sql
SELECT
  id,
  video_id,
  title,
  channel_name,
  fit_score,
  summary,
  why_it_matters,
  url
FROM watch_candidates
WHERE used_in_issue_date IS NULL
  AND discarded = false
ORDER BY fit_score DESC, published_at DESC
LIMIT 5;
```

### Mark Video as Selected

```sql
UPDATE watch_candidates
SET used_in_issue_date = CURRENT_DATE
WHERE id = 'paste-video-uuid-here';
```

### Discard Video

```sql
UPDATE watch_candidates
SET discarded = true
WHERE id = 'paste-video-uuid-here';
```

### View Statistics

```sql
SELECT
  COUNT(*) as total_videos,
  COUNT(*) FILTER (WHERE used_in_issue_date IS NOT NULL) as used,
  COUNT(*) FILTER (WHERE discarded = true) as discarded,
  COUNT(*) FILTER (WHERE used_in_issue_date IS NULL AND discarded = false) as available,
  ROUND(AVG(fit_score), 2) as avg_score
FROM watch_candidates;
```

### Check for Duplicates

```sql
-- Should return 0 rows
SELECT video_id, COUNT(*)
FROM watch_candidates
GROUP BY video_id
HAVING COUNT(*) > 1;
```

## REST API Quick Reference

All endpoints are auto-generated by Supabase.

### Base URL
```
https://your-project-id.supabase.co/rest/v1
```

### Get Shortlist (via REST)
```
GET /watch_candidates?used_in_issue_date=is.null&discarded=eq.false&order=fit_score.desc&limit=5

Headers:
  apikey: your-service-key
  Authorization: Bearer your-service-key
```

### Upsert Video (via REST)
```
POST /watch_candidates?on_conflict=video_id

Headers:
  apikey: your-service-key
  Authorization: Bearer your-service-key
  Content-Type: application/json
  Prefer: resolution=merge-duplicates

Body: { video_id, title, summary, ... }
```

### Mark as Selected (via REST)
```
PATCH /watch_candidates?id=eq.UUID

Headers:
  apikey: your-service-key
  Authorization: Bearer your-service-key
  Content-Type: application/json

Body: { "used_in_issue_date": "2024-01-15" }
```

## Troubleshooting

### Problem: Can't connect to Supabase
- Verify your `SUPABASE_URL` is correct
- Check API key includes both `apikey` header AND `Authorization` header
- Test with curl:
  ```bash
  curl "$SUPABASE_URL/rest/v1/watch_candidates?limit=1" \
    -H "apikey: $SUPABASE_SERVICE_KEY" \
    -H "Authorization: Bearer $SUPABASE_SERVICE_KEY"
  ```

### Problem: Duplicate videos in database
- Add `?on_conflict=video_id` to your POST URL in n8n
- Run deduplication query:
  ```sql
  DELETE FROM watch_candidates a USING (
    SELECT MIN(id) as id, video_id
    FROM watch_candidates
    GROUP BY video_id
    HAVING COUNT(*) > 1
  ) b
  WHERE a.video_id = b.video_id AND a.id != b.id;
  ```

### Problem: fit_score validation error
- Ensure fit_score is an INTEGER between 1-10
- Add validation in n8n before sending
